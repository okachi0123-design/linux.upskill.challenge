## What I Learned
- `locate` は `updatedb` で作られたDBを検索するため高速だが、最新性に欠ける。Ubuntu系では `plocate` のインストールが必要
- `sudo locate` を使うと権限のあるファイルも検索可能
- `find` はリアルタイムでファイルシステムを直接走査する
- `find` は名前 (`-name`) だけでなく、更新時刻 (`-mtime`) など多様な条件で検索できる
- `grep -R` でディレクトリを再帰的に検索でき、設定ファイル探しに強い (例: `/etc` 配下から `PermitRootLogin` を発見)
- `-i` オプションは大文字小文字を区別しない検索 (typo対策にも有効)
- `which` で実行コマンドのパスを確認できる。`$PATH` に基づいて `/bin` `/usr/bin` などから探される
- `sudo !!` は直前のコマンドに `sudo` を付けて再実行する便利技
- 圧縮ログ (`.gz`) には `zless` / `zgrep` が使える

## Commands Used
```bash
# which - コマンドのパスを確認
which vim                                          # vimバイナリの場所を表示

# find - リアルタイム検索
find /var -name access.log                         # 名前指定でファイル検索 (権限ないと一部エラー)
sudo find /var -name access.log                    # 権限のあるディレクトリも検索 → /var/log/apache2/access.log を発見
find /home -mtime -3                               # /home 配下で過去3日以内に更新されたファイル
man find                                           # findのマニュアル参照

# locate - DB型の高速検索
locate access.log                                  # plocate未インストールでエラー
sudo apt install plocate                           # plocateを導入
locate access.log                                  # DB未構築のため何も出ない
sudo updatedb                                      # locate用DBを構築
locate access.log                                  # それでも見つからない (権限問題)
cat /etc/updatedb.conf | grep -i prune             # 除外設定を確認 → /var/log は除外されていなかった
sudo locate access.log                             # rootとして実行 → 発見

# grep -R - 再帰的な文字列検索
sudo grep -R -i "PwemitRootLogin" /etc/*           # typoしたバージョン (何も出ない)
sudo grep -R -i "PermitRootLogin" /etc/*           # 正しいスペルで /etc 配下を再帰検索

# その他
history                                            # コマンド履歴の確認
sudo !!                                            # 直前のコマンドにsudoを付けて再実行
```

## Where I Got Stuck
- `locate access.log` を実行しても結果が出ない → `sudo updatedb` で再構築しても出ない → `/etc/updatedb.conf` の `PRUNEPATHS` を確認したが `/var/log` は除外対象ではなかった → 真の原因は **plocateのプライバシー保護仕様**(ユーザーが読み取り権限を持たないファイルは検索結果に表示しない)。`sudo locate access.log` で解決
- 最初に "PwemitRootLogin" とtypoして検索結果がゼロになった → `grep -i` で大小区別なし検索にしても、スペル自体が違うとヒットしないことを実感
