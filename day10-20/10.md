## What I Learned

- `at` は1回限りの未来のタスクをスケジュールするコマンド
- `cron` は時刻ベースで繰り返しタスクを実行するデーモン
- ユーザー個別のcrontabは `crontab -e` で編集、`crontab -l` で表示、`crontab -r` で削除
- ユーザーcrontabの実体は `/var/spool/cron/crontabs/` 配下に保存される
- システム全体のcrontabは `/etc/crontab` にあり、**ユーザー名フィールド**がある点が個人crontabと違う
- crontabの書式は「分 時 日 月 曜日 コマンド」(システムcrontabはユーザー名が間に入る)
- 曜日は 0-6 (0=日曜) または `sun,mon,tue,wed,thu,fri,sat` で指定可能
- `*` はワイルドカード、`,` で複数値、`-` で範囲、`/` で周期を表現できる
- `@reboot` は再起動直後に1回実行される特殊指定
- `/etc/cron.{hourly,daily,weekly,monthly}/` にスクリプトを置くだけで定期実行できる
- `$(date)` でコマンド実行結果を埋め込め、`date -I` でISO形式の日付が取れる
- `tar -czvf` で圧縮アーカイブを作成（c=create, z=gzip, v=verbose, f=file）
- `find /path -mtime +7 -delete` で7日より古いファイルを削除でき、バックアップ管理に使える
- systemdの `timer` も cron の代替として使える(`systemctl list-timers` で確認)

## Commands Used

```bash
# cronの基本操作
man cron                                      # cronのマニュアルを読む
crontab -l                                    # 自分のcrontabを表示
crontab -e                                    # 自分のcrontabを編集
crontab -r                                    # 自分のcrontabを削除

# システムcrontabの編集
vim /etc/crontab                              # 権限なしで開いてしまう (読み取り専用)
sudo vim /etc/crontab                         # rootで編集 (正しい方法)

# 動作確認・状態チェック
sudo systemctl status cron                    # cronサービスが動いているか確認
sudo journalctl -u cron --since "2 hours ago" | tail -30   # cronのログをsystemd経由で確認
sudo grep CRON /var/log/syslog | tail -20     # cronのログをsyslogから確認

```bash
# /etc/crontab に追加した実際のジョブ
41 *    * * *     root   echo $(date) "Sched message" >> /home/murabito/cronmessages   # 毎時41分に実行
47 13   * * sat   root   echo $(date) "TGIF" >> /home/murabito/cronmessages            # 毎週土曜13:47に実行
@reboot           root   echo $(date) "Good reboot!" >> /home/murabito/cronmessages    # 起動直後に1回実行
```

## Where I Got Stuck

- ジョブ実行後、ターミナルのプロンプトに 30 * * * * root echo ... という古い行が表示され続けた → ターミナル上の文字をマウスで選択した際にLinuxの「プライマリ選択バッファ」に入り、中ボタンクリック等で意図せずシェルに貼り付けられていた(history で確認するとcrontabの行が履歴に残っていた) → Ctrl+U で入力中の文字をクリア or Ctrl+C でキャンセル、最終的に sudo reboot でクリーンな状態に戻して解決。今後はターミナル内のテキストをマウスで選択しないよう注意
- sudo cat /etc/crontab の出力をマウスで選択したら複数行がシェルに流れ込み command not found が連発 → Ctrl+C 連打 → ターミナル閉じて開き直す or sudo reboot。長い出力を見るときは sudo cat /etc/crontab | less(qで抜ける)が安全
