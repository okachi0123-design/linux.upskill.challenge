## Today's Theme
The infamous "grep" and other text processors
## What I Learned
- `tail -f`で**ログをリアルタイム監視**できる(Apache の動作確認やデバッグに必須)
- パイプ `|` でコマンドを連結し、出力を次のコマンドの入力に渡せる
  - 例: `cat file | grep "word"` は `grep "word" file` と同じ
  - 複数の `grep` を重ねれば検索を絞り込める
- `grep` の便利オプション
  - `-v`: 指定パターンを**除外**(反転マッチ)
  - `-a`: バイナリ判定されたファイルもテキストとして扱う
  - `-i`: 大文字小文字を区別しない
  - `-E`: 拡張正規表現(`+`、`{}`などをエスケープ不要で使える)
- `cut` でフィールド単位の抽出ができる
  - `-d` で区切り文字、`-f` でフィールド番号を指定
  - `-f 10-` のように範囲指定も可能(10番目以降)
- リダイレクト `>` でコマンド出力をファイルに保存できる
  - `>`: 上書き(無ければ作成)
  - `>>`: 追記
- `sort` と `uniq` を組み合わせると重複除去ができる
  - `uniq` は連続する重複しか除去しないので、必ず `sort` してから使う
- `/var/log/auth.log` が無い場合、minimal版Ubuntuの可能性
  - `sudo apt install rsyslog` で生成されるようになる
- `auth.log`が"binary file matches"と判定されることがある
  - `grep -a` でテキスト扱いさせて中身を表示
## Commands Used
```bash
# ファイル全体の表示
cat /var/log/apache2/access.log
# ページャーで表示(less内で gg / GG / / / n / N / q が使える)
less /var/log/apache2/access.log
less /var/log/auth.log                    # ログイン・sudo履歴の確認
# 先頭・末尾だけ見る
head /var/log/apache2/access.log          # 先頭10行
tail /var/log/apache2/access.log          # 末尾10行
tail -n 50 /var/log/apache2/access.log    # 末尾50行
# リアルタイム監視
tail -f /var/log/apache2/access.log       # 新しい行が追記されると即表示
tail -F /var/log/apache2/access.log       # ログローテーションにも追従
# パイプで grep に渡す
cat /var/log/auth.log | grep "authenticating"
grep "authenticating" /var/log/auth.log   # 上と同じ(こちらが簡潔)
# 複数の grep で絞り込み
grep "authenticating" /var/log/auth.log | grep "root"
# cut でフィールド抽出(スペース区切りの10番目以降)
grep "authenticating" /var/log/auth.log | grep "root" | cut -f 10- -d" "
# -v で除外(root 以外のログイン試行)
grep "authenticating" /var/log/auth.log | grep -v "root" | cut -f 10- -d" "
# binary file matches と出たとき
sudo grep -a "root" /var/log/auth.log
# リダイレクト
ls -ltr > listing.txt                     # 上書き
echo "log" >> listing.txt                 # 追記
# 攻撃IP一覧をファイルに保存(本日の課題)
sudo grep -a "authenticating" /var/log/auth.log \
  | grep "root" \
  | cut -f 10- -d" " > ~/attackers.txt
# IPアドレスだけ抽出してユニーク化(発展課題)
sudo grep -a "authenticating" /var/log/auth.log \
  | grep "root" \
  | grep -oE "([0-9]{1,3}\.){3}[0-9]{1,3}" \
  | sort \
  | uniq
# auth.log が無いとき
sudo apt install rsyslog                  # 数分待つと生成される
# awk と sed の例(発展)
awk '{print $1, $9}' /var/log/apache2/access.log   # 1番目と9番目のフィールドだけ表示
sed 's/error/ERROR/g' file.log                     # error を ERROR に置換
```
## Where I Got Stuck
- `grep`で `/var/log/auth.log` を検索したら `binary file matches` と表示されて中身が出なかった
  → ファイルに非印字文字が混じると `grep` がバイナリと誤判定する
  → `grep -a` を付ければテキストとして扱って中身を表示してくれる
- `sudo grep -a "root" /var/log/auth.log | grep "[0-9]\{1,3\}\..."` が**何も返ってこなかった**
  → ログが空 or ローテーションされて `auth.log.1` や `auth.log.2.gz` に移動している可能性
  → `sudo wc -l /var/log/auth.log` でファイルに中身があるか確認、`zgrep` で過去ログも検索できる
- 正規表現の `\{1,3\}` がエスケープだらけで読みにくかった
  → `grep -E "([0-9]{1,3}\.){3}[0-9]{1,3}"` のように `-E` を付けるとエスケープ不要で書ける
- `tail` と `tac` を混同した
  → `tail`: 末尾を表示
  → `tac`: 行の順序を逆にする(`cat` を逆から綴ったコマンド)
  → さらに `rev` は**各行内の文字を逆順**にする別物
