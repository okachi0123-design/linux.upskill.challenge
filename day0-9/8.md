# Day8
## Date
2026-04-30
## Today's Theme
The infamous "grep" and other text processors
## What I Learned
- `tail -f`でログをリアルタイム監視できる
- `grep -v` で除外、`grep -a` でバイナリ判定回避、`grep -E` で拡張正規表現
- `cut -d -f` でフィールド単位の抽出
- `head`（先頭表示）/ `tail`(末尾表示) / `tac`(行順を逆) / `rev`(行内文字を逆) 
## Commands Used
```bash
# 表示系
cat /var/log/apache2/access.log           # 全部表示
less /var/log/apache2/access.log          # ページャー(gg/GG/?n/Nが使える)
head file                                 # 先頭10行
tail file                                 # 末尾10行
tail -f /var/log/apache2/access.log       # リアルタイム監視
tail -F file                              # ローテーションにも追従
# grep の基本
grep "root" /var/log/auth.log             # 検索
grep -a "root" /var/log/auth.log          # バイナリ判定回避
grep -v "root" file                       # 除外
grep -i "error" file                      # 大文字小文字無視
grep -E "([0-9]{1,3}\.){3}[0-9]{1,3}" file  # 拡張正規表現
# パイプで連結
grep "authenticating" /var/log/auth.log | grep "root"
grep "authenticating" /var/log/auth.log | grep -v "root" | cut -f 10- -d" "
# リダイレクト
ls -ltr > listing.txt                     # 上書き
echo "log" >> listing.txt                 # 追記
# 本日の課題: 攻撃IP一覧をファイルに保存
sudo grep -a "authenticating" /var/log/auth.log \
  | grep "root" | cut -f 10- -d" " > ~/attackers.txt
# 発展: IPアドレスだけ抽出してユニーク化
sudo grep -a "authenticating" /var/log/auth.log | grep "root" \
  | grep -oE "([0-9]{1,3}\.){3}[0-9]{1,3}" | sort | uniq
# awk と sed
awk '{print $1, $9}' access.log           # 1番目と9番目のフィールドを表示
sed 's/error/ERROR/g' file.log            # 文字列置換
```
## Where I Got Stuck
- `grep`で `binary file matches` と出て中身が見えなかった → `grep -a` でテキスト扱い
- 攻撃IP抽出コマンドが何も返さなかった → ログがローテーションされた可能性、`zgrep` で過去ログ(`auth.log.1`, `auth.log.*.gz`)も検索する
- `\{1,3\}` のエスケープが面倒 → `grep -E` で `{1,3}` のまま書ける
## Insights & Reflections
ログ解析の基本パターン(`tail -f`で監視 → `grep`で絞り込み → `cut`で抽出 → `sort | uniq`で集計)が身についた
