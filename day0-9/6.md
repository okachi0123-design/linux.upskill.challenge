## What I Learned
- Linuxではテキストファイルが中心的存在で、それを編集する力はsysadminの基本スキル
- 初心者向けには`nano`や`pico`があるが、プロのsysadminは`vi`/`vim`を使う
- `vi`はBill Joyが1970年代半ばに作ったエディタで、`vim`（Vi IMproved）はその直系の改良版
- 多くのシステムでは`vi`コマンドを実行すると実際は`vim`が起動する（`vi --version`で確認可能）
- `vi`はSingle Unix SpecificationやPOSIXで必須とされており、どんなUnix/Linux環境でも必ず入っている
- Vimには2つのモードがある：**ノーマルモード**（コマンド入力用）と**インサートモード**（テキスト入力用）
- 画面上に現在のモードを示す表示はほぼないため、わからなくなったら`Esc`を2回押せばノーマルモードに戻れる
- 初心者がまず覚えるべき3つの基本：
  1. `Esc` `Esc` でノーマルモードに戻る
  2. `:q!` で変更を保存せずに終了
  3. `u` で直前の操作を取り消す
- `33dd`のように数字＋コマンドで「33行削除」のような連続操作ができる
- `vimtutor`コマンドで公式チュートリアル（約30分）が体験できる。1〜2週間毎日やると基礎が身につく
- `~/.vimrc`でVimをカスタマイズ可能

## Commands Used
| コマンド | 説明 |
|----------|------|
| `vim ファイル名` | ファイルをVimで開く |
| `vi --version` | viがvimにsymlinkされているか確認 |
| `Esc` `Esc` | ノーマルモードに戻る |
| `i` | インサートモードに入る |
| `h` `j` `k` `l` | 左・下・上・右に移動（伝統的なvi方式） |
| `gg` | ファイルの先頭に移動 |
| `G` | ファイルの末尾に移動 |
| `/検索語` | 検索（`n`で次の結果へ） |
| `n` | 次の検索結果に移動 |
| `dd` | 1行削除（カット） |
| `33dd` | 33行削除 |
| `P` | カーソルの前にペースト |
| `p` | カーソルの後にペースト |
| `u` | アンドゥ（元に戻す） |
| `Ctrl + r` | リドゥ（やり直し） |
| `:w` | 保存（Vimは終了しない） |
| `:wq` | 保存して終了 |
| `:q!` | 変更を破棄して終了 |
| `:%s/old/new/gc` | ファイル全体を確認しながら置換 |
| `vimtutor` | Vim公式チュートリアル起動 |

## Where I Got Stuck
- `Ctrl + r`を押そうとして`Shift + r`を押してしまい、リプレースモードに入ってしまった→ `Esc`で解決
- `:%s/port/fork/gc`の`c`オプションで確認プロンプトが出た際の選択肢（y/n/a/q/l）の意味がわからなかった→ 一つずつ調べて理解
  - `y`=置換 / `n`=スキップ / `a`=残り全部置換 / `q`=終了 / `l`=この箇所だけ置換して終了
- `/`で検索後に`n`を押したら`E35: No previous regular expression`エラーが出た→ 先に検索パターンを入力する必要があると理解
- 最初は画面に「今どのモードか」が表示されないことに戸惑った→ ノーマル（コマンド）モードで、別のモードからはESCで戻れる
- インサートモードからビジュアルモードに直接行こうとしてうまくいかなかった→ 必ずノーマルモードを経由する必要があると学んだ
