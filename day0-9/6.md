## What I Learned
- Linuxでテキストファイル編集は重要な管理スキル。プロは`vi`/`vim`を使う
- `vi`は1970年代にBill Joyが開発、`vim`は"Vi IMproved"の略でその後継
- 多くのシステムでは`vi`コマンドが`vim`へのシンボリックリンクになっている
- vimには2つのモードがあり、見た目では区別しにくい
  - **ノーマルモード**: コマンド入力用
  - **インサートモード**: 文字入力用
- 初心者の3つの基本ルール
  - `Esc` `Esc` で必ずノーマルモードに戻れる
  - ノーマルモードで`:q!`なら保存せず終了できる
  - ノーマルモードで`u`なら直前の操作を取り消せる
- `vi`はUnix/Linuxに必ず入っている標準エディタ(POSIX規格で要求されている)
- `nano`は使いやすいが、他人のサーバーでは入っていない・インストール権限がないことも多い
- `vimtutor`コマンドで公式チュートリアルが起動する(30分程度)

## Commands Used
```bash
# vimのバージョン確認(viがvimにリンクされているか)
vi --version

# vimがない場合の対処
echo "alias vim='vi'" >> ~/.bashrc    # viをvimとして使う
source ~/.bashrc                       # 設定を反映
sudo apt install vim                   # 正式にインストール

# 練習用ファイル準備
cp -v /etc/services testfile           # コピーで練習用を作る
vim testfile                           # vimで開く

# vimのチュートリアル
vimtutor                               # 公式チュートリアル起動
```

```vim
" === ノーマルモード操作 ===

" カーソル移動(矢印キーが使えない環境でも動く)
h     " 左
j     " 下
k     " 上
l     " 右

" ファイル内移動
gg    " ファイル先頭へ
G     " ファイル末尾へ
<num>G " 指定行へジャンプ

" 検索
/word  " 前方検索
?word  " 後方検索
n      " 次の検索結果
N      " 前の検索結果

" 削除・コピペ
dd     " 1行削除(同時に切り取り)
33dd   " 33行削除
P      " 削除した内容を貼り付け(カーソル前)
p      " 削除した内容を貼り付け(カーソル後)

" 取り消し
u      " アンドゥ(直前の操作を取り消し)

" インサートモードへ
i      " カーソル位置から挿入

" 終了系
:w     " 保存
:q     " 終了
:wq    " 保存して終了
:q!    " 保存せず強制終了
```

## Insights & Reflections
- 最初は「なぜこんな複雑なエディタを使うのか」と思ったが、理由は明確：`vi`はあらゆるUnix/Linux環境に必ず入っているから。他人のサーバーで作業する時、`nano`がない環境でも困らない
- 「モード切り替え」という概念が独特。普通のエディタ(メモ帳など)は常に文字入力モードだが、vimでは「コマンド」と「入力」を明確に分ける
- 一見不便に見える`hjkl`での移動も、ホームポジションから手を離さずに済むので、慣れると速い
- 「Vimをどう終了するか」が世界中で笑い話になるくらい初心者を困らせる(`how-to-exit-vim`という記事まである)。`:q!`を最初に覚えるのが鉄則
- `vimtutor`を毎日1〜2週間続けると基礎が固まると書かれていた。一度きりではなく繰り返し練習するのが大事
- プロのシステム管理者を目指すなら、これからの編集は全部vimで行う「コミット」が必要。今のうちに慣れてしまうのが一番
