## What I Learned
- Apache2はWebサーバーソフトウェアで、apt install apache2　でWebサイトができる
- VMにApache2をインストールすると、ホストPCのブラウザから`http://localhost:[ホストポート]`でアクセスできる(apache用ポートフォワーディング必要)
- VM内では`curl http://localhost`でApacheの動作確認ができる
- `/var/www/html/index.html`がデフォルトの公開ページ(sudo vim　で内容変更できる)
- Apache2の設定ファイル構造(Debian/Ubuntu系)
  - `/etc/apache2/apache2.conf` がメイン設定ファイル
  - `conf-available/` に全ての設定ファイルを置き、`conf-enabled/` にシンボリックリンクで有効化
  - 同じ仕組みで `mods-available/`/`mods-enabled/`、`sites-available/`/`sites-enabled/` も存在
- `Include`/`IncludeOptional`で他の.confファイルを読み込む仕組み
  - `Include`: 必ず読み込む(無いとエラー)
  - `IncludeOptional`: あれば読み込む(無くてもOK)
- 設定ファイルはすべてローカル(VM内)から読み込まれる、インターネット不要
- ログファイルは`/var/log/apache2/`にあり、root権限が必要
- `access.log`にはアクセス記録、`error.log`にはエラー情報が記録される

## Commands Used
```bash
# Apache2のインストールと操作
sudo apt install apache2              # インストール
sudo systemctl status apache2         # 状態確認
sudo systemctl start apache2          # 起動
sudo systemctl reload apache2         # 設定再読み込み
sudo systemctl restart apache2        # 再起動

# 動作確認
curl http://localhost                 # VM内から動作確認

# 設定ファイルの確認
ls /etc/apache2/                      # Apache設定ディレクトリ
cat /etc/apache2/apache2.conf         # メイン設定ファイル
ls /etc/apache2/conf-enabled/         # 有効な追加設定
ls /etc/apache2/mods-enabled/         # 有効なモジュール
ls /etc/apache2/sites-enabled/        # 有効なサイト設定

# ページの編集
sudo vim /var/www/html/index.html     # トップページ編集(sudo必須)

# vim操作(おさらい)
# ggdG    : ファイル全体を削除
# :%d     : ファイル全体を削除(コマンドラインモード)
# i       : 編集モード
# Esc     : ノーマルモードに戻る
# :wq     : 保存して終了
# :q!     : 保存せず終了

# ログの確認
sudo tail /var/log/apache2/access.log     # 末尾10行を表示
sudo tail -f /var/log/apache2/access.log  # リアルタイム監視
sudo less /var/log/apache2/access.log     # ページャーで閲覧
sudo cat /var/log/apache2/error.log       # エラーログ確認
```

## Where I Got Stuck
- `http://localhost`にアクセスして`ERR_INVALID_HTTP_RESPONSE`が出た
  →　SSH用のポートフォワーディング設定でアクセスしたが、Webアクセスには使えないことが分かった
- `vim /var/www/html/index.html`で開いたら`[readonly]`表示
  → ファイル所有者がrootなので、`sudo vim`で開き直す必要がある
  → 編集自体はバッファ上でできるが保存できない
- `apache2.conf`を変更すると他の.confファイルも自動変更されるのか混乱した
  → ファイル同士は独立。Apacheが起動時に全部を合体させて読むだけ
  → どのファイルを編集しても他の中身は変わらないが、Apache全体の動作には影響する
