## What I Learned
### SSH接続
- NATポートフォワーディングでアクセスを可能にする
- SSHサービスの有効化・起動でSSH接続を受け入れる
- net-toolsのインストール方法（ifconfig、netstat）
- ネットワーク・システム情報を確認するコマンド群
## Commands Used
```bash
# SSHを有効化
sudo systemctl enable ssh

# SSHを起動
sudo systemctl start ssh

# WindowsのPowerShellからSSH接続
ssh user@ipadress -p portnunber
# net-toolsをインストール
sudo apt install net-tools

# ネットワーク関連
ifconfig                  # ネットワークインターフェース情報
netstat -tuln | grep 22  # SSH(port22)が動いているか確認
netstat -i               # ネットワークインターフェース統計
ip addr                  # IPアドレス確認（モダンコマンド）
sudo iftop -i eth0       # リアルタイムネットワークトラフィック監視
ifstat                   # ネットワーク帯域使用量をリアルタイム表示

# ディスク・ハードウェア関連
df -h                    # ディスク使用量
lsusb                    # 接続中のUSBデバイス一覧
lspci                    # PCIデバイス一覧
lsblk                    # ブロックデバイス一覧
lscpu                    # CPU情報

# システム情報
uname -a                 # カーネル情報
lsb_release -a          # Ubuntuバージョン情報
whoami                   # 現在のユーザー確認

# リソース監視
htop                     # インタラクティブなプロセス監視
top                      # プロセス監視
vmstat                   # 仮想メモリ・CPU統計
free -h                  # メモリ使用量
```
## Where I Got Stuck
- VirtualBoxのブリッジアダプター設定でVMが起動しなくなった
  - 原因：Hyper-V有効化・Wi-FiアダプターのMAC制限・WSL2との競合
  - 解決：NATポートフォワーディングに切り替え
- ubuntu-serverが黒画面のまま起動しなかった
  - 解決：電源オフ → 再起動
  - GUIなしで起動
- SSH接続がConnection refusedになった
  - SSHがdisabledだった → `sudo systemctl enable ssh`
  - 間違ったターミナルからSSHを実行していた → PowerShellから実行
  - SSHがactiveになっていなかった → `sudo systemctl start ssh
  - NATのままで接続しようとしていた　→　NATポートフォワーディング
