## What I Learned
- `more`は1画面ずつ表示するシンプルなページャー
- `less`は`more`の上位互換で、前後スクロール・検索が自由にできる
- 大きなファイルでも`less`は必要な分だけ読み込むので高速
- `history`コマンドはメモリ上の履歴を表示
- `~/.bash_history`はディスクに保存された履歴
- シェル終了時にメモリ→ファイルに書き出される
- 履歴の活用
  - `↑`キーで過去のコマンドを呼び出し
  - `!20`で履歴20番目を再実行
  - `Ctrl + R`でインクリメンタル検索
## Commands Used
```bash
# ページャー (less)
less <file>                           # ファイルを1画面ずつ表示
# less内操作
# スペース / f  : 次の画面へ
# b             : 前の画面へ
# Enter / ↓ / e : 1行下へ
# ↑ / y         : 1行上へ
# d             : 半画面下へ
# u             : 半画面上へ
# g             : ファイル先頭へ
# G             : ファイル末尾へ
# <num>g        : 指定行へジャンプ
# /<word>       : 前方検索
# ?<word>       : 後方検索
# n             : 次の検索結果
# N             : 前の検索結果
# =             : 現在位置(行番号など)を表示
# h             : ヘルプ表示
# v             : エディタで開く
# q             : 終了
# ページャー (more)
more <file>                           # シンプルなページャー
more -10 <file>                       # 1画面を10行で表示
more +50 <file>                       # 50行目から表示
# more内操作
# スペース      : 次の画面へ
# Enter         : 1行下へ
# b             : 前の画面に戻る(限定的)
# /<word>       : 前方検索
# n             : 次の検索結果
# =             : 現在の行番号を表示
# q             : 終了
# 履歴操作
history                               # メモリ上の履歴を表示
!<num>                                # 履歴n番目を再実行
!!                                    # 直前のコマンドを再実行
# Ctrl + R で履歴インクリメンタル検索
# 隠しファイル
less ~/.bashrc                        # シェル設定ファイル
less ~/.bash_history                  # コマンド履歴ファイル
```
## Where I Got Stuck
- `less ~/.bash_history`で「may be a binary file」警告が出た
  → 原因は急なシャットダウン時にNULバイトが混入していたため
- `history`コマンドと`~/.bash_history`ファイルの違い
  → メモリ上の履歴とディスク上の履歴は別物で、終了時に同期される
